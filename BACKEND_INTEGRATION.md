## How It Works

### 1. Form Submission Flow (Home.jsx)
```
User enters name → Clicks "Generate Wish"
         ↓
handleGenerate() called
         ↓
recordVisitor(name) → API call (fire-and-forget)
         ↓
navigate() → User sees greeting page immediately
         ↓
Backend saves name to MongoDB in background
```

### 2. Direct URL Access Flow (Greeting.jsx)
```
User visits /name=Ali
         ↓
Component mounts
         ↓
parseNameFromPath() extracts "Ali"
         ↓
useEffect triggers recordVisitor("Ali") → API call
         ↓
Component renders greeting page
         ↓
Backend saves name to MongoDB in background
```

## Fire-and-Forget Implementation

The API calls are designed to NOT block the UI:

```javascript
// In visitorAPI.js
export const recordVisitor = async (name) => {
  try {
    // Make request WITHOUT await in the caller
    fetch(`${API_BASE_URL}/api/visitors`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ name }),
    })
    // No error handling shown to user
  } catch (error) {
    // Silently fail - log only
    console.error('Error recording visitor:', error)
  }
}
```

In Home.jsx:
```javascript
const handleGenerate = (enteredName) => {
  setName(enteredName)
  recordVisitor(enteredName)  // Don't await
  navigate(`/name=${encoded}`) // Navigate immediately
}
```

## API Endpoint Reference

### POST /api/visitors
**Purpose:** Store a visitor's name

**Request:**
```bash
curl -X POST http://localhost:5000/api/visitors \
  -H "Content-Type: application/json" \
  -d '{"name": "Imran Khan"}'
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Visitor recorded successfully",
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    "name": "Imran Khan",
    "createdAt": "2024-03-21T10:30:00.000Z",
    "updatedAt": "2024-03-21T10:30:00.000Z"
  }
}
```

**Response (Error):**
```json
{
  "success": false,
  "message": "An error occurred" // Generic in production
}
```

### GET /api/visitors
**Purpose:** Retrieve all visitor records

**Request:**
```bash

```

**Response:**

## Key Design Decisions

### 1. Fire-and-Forget API Calls
- **Why:** Prevents UI lag, provides smooth user experience
- **How:** recordVisitor() doesn't return a promise to caller
- **Trade-off:** User doesn't see if save succeeded

### 2. Allow Duplicate Names
- **Why:** Track multiple visits from same person
- **How:** No unique constraint on name field
- **Use Case:** Count total greetings sent

### 3. Silent Error Handling
- **Why:** Don't confuse users with backend errors
- **How:** Catch errors and log only
- **Production:** Generic error messages prevent info leaks

### 4. Trim Whitespace
- **Why:** Consistent data storage
- **How:** Mongoose schema applies trim
- **Example:** "  Ali  " → "Ali"

## Common Issues & Solutions

### API calls failing silently
**Symptom:** Names not appearing in MongoDB
**Solution:**
1. Check if backend is running (`npm run dev` in server folder)
2. Verify VITE_API_URL in client/.env matches server PORT
3. Check browser Network tab for failed requests
4. Check server console for error logs

### CORS Errors
**Symptom:** "Access to XMLHttpRequest blocked by CORS"
**Solution:**
1. Verify server has CORS middleware enabled
2. Check CORS_ORIGIN in server/.env
3. Use `*` for development: `CORS_ORIGIN=*`

### MongoDB not connecting
**Symptom:** "Error connecting to MongoDB"
**Solution:**
1. Verify MONGODB_URI in server/.env
2. Check IP whitelist in MongoDB Atlas
3. Ensure credentials are correct
4. Test: `node -e "require('./config/database').default()"`

### Port conflicts
**Symptom:** "Error: listen EADDRINUSE :::5000"
**Solution:**
1. Change PORT in server/.env
2. Or kill process: `lsof -i :5000 | grep LISTEN | awk '{print $2}' | xargs kill -9`
